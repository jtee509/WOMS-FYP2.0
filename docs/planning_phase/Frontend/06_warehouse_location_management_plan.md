# Planning Document: Warehouse Location Management (Frontend)
**Module:** Settings → Warehouse → Location Management
**Status:** Implemented (v0.5.40)
**Author:** Claude Code (retrospective — created 2026-03-09)

---

## 1. Overview & Goals

The Warehouse Location Management module gives warehouse administrators a complete UI for:

- Viewing all physical locations grouped by **section** (e.g. A, B, C) inside an accordion tree
- **Inline editing** of any location's code, description, aisle, row, level, position, and status
- **Bulk operations** — activate, deactivate, or soft-delete multiple locations at once
- **Pattern Generator** — a wizard that generates location codes by combining section / aisle / row / level / position ranges, previews the output, and commits it to the database
- **CSV / Excel Import** — drag-and-drop upload with SheetJS validation and a column-mapping step
- **Soft delete only** — no hard deletes are ever exposed in the UI; deleted records remain in the DB

This feature was implemented as a sub-section of `SettingsPage` under the **Warehouse** tab and is tightly coupled to a per-warehouse context (the active warehouse is chosen via `WarehouseSelector`).

## 2. Scope

### In-scope
| Area | Detail |
|---|---|
| Location list view | Accordion grouped by section, sortable, paginated |
| Inline edit | All string/number fields editable per-row; save on confirm |
| Bulk selection | Checkbox per row + header checkbox; bulk activate/deactivate/delete |
| Pattern Generator | Wizard panel (left) + Preview Tree (right), resizable split pane |
| Existing-section populate | Selecting an existing section in the wizard auto-loads its current locations into the right panel in editable (`directEdit`) mode |
| Draft preview | Generated locations shown below existing ones with a dashed separator |
| CSV Import | SheetJS parse → column validation → progress bar → commit |
| Soft delete | Frontend always POSTs to the soft-delete endpoint; no hard-delete button |

### Out-of-scope (deferred)
- Location bulk reassignment (move locations between warehouses)
- Barcode / QR label printing
- Occupancy heat-map view

## 4. State Architecture

### LocationManagementSection — key state

| State | Type | Purpose |
|---|---|---|
| `locations` | `InventoryLocation[]` | Flat list fetched from API |
| `grouped` | `Map<string, InventoryLocation[]>` | Derived: locations keyed by section |
| `pagedKeys` | `string[]` | Section keys visible on current page |
| `selectedIds` | `Set<number>` | Currently ticked location IDs (shared between normal list and generator right panel) |
| `editingId` | `number \| null` | ID of the row currently in inline-edit mode |
| `editDraft` | `Partial<InventoryLocation>` | Staging buffer for in-progress cell edits |
| `showGenerator` | `boolean` | Whether the split-pane generator is open |
| `selectedGenSection` | `string` | Section the wizard has selected (existing mode); drives right-panel content |
| `previewLocs` | `PreviewLocation[]` | Draft locations generated by the wizard |
| `leftPct` | `number` | Left panel width as % of container (resizable split pane) |
| `confirmBatch` | `'delete' \| null` | Controls the delete-confirm dialog |
| `batchDeleting` | `boolean` | Loading state for batch operations |

### BulkCreationWizard — key state

| State | Type | Purpose |
|---|---|---|
| `sectionMode` | `'new' \| 'existing'` | Whether the user is creating a new section or extending an existing one |
| `existingSec` | `string` | Selected existing section name |
| `newSectionName` | `string` | User-typed new section name |
| `levels` | `Record<Level, LevelConfig>` | Toggle + range config for each level |
| `fixedSection` | `string` (derived) | Resolves to `existingSec` or `newSectionName.trim()` |

## 5. Key Design Decisions

### 5.1 directEdit Mode
**Problem:** In generator context, the Preview Tree needs to be immediately editable without an extra pencil-click UX step.
**Decision:** Added `directEdit?: boolean` prop to `SectionAccordion`. When true:
- Cells show `cursor-text` and a hover border on hover
- Clicking a cell calls `onStartEdit(loc.id)` immediately
- The pencil icon is always visible (muted) rather than hidden behind `group-hover`

### 5.2 selectedGenSection — Right Panel Priority
**Problem:** When user picks an existing section in the wizard AND also generates a draft preview, which should the right panel show?
**Decision:** `selectedGenSection` takes highest priority:
1. If `selectedGenSection` is set → show that section's existing locations (editable) + draft preview stacked below with a dashed divider
2. Else if `previewLocs.length > 0` → show draft-only PreviewAccordion
3. Else if `loading` → spinner
4. Else → show all locations paginated

### 5.3 fixedSection Derivation
When `sectionMode === 'existing'`: `fixedSection = existingSec`
When `sectionMode === 'new'`: `fixedSection = newSectionName.trim()`

This ensures generated location codes always include the correct section prefix regardless of mode.

### 5.4 anyEnabled Guard
When a `fixedSection` exists, the section level is logically already satisfied — so the "Generate" button only checks that at least one sub-level (aisle/row/level/position) is enabled, preventing accidental empty-section-only generation.

### 5.5 Resizable Split Pane
Uses a drag handle with `mousedown` → `mousemove` on `window` → `useRef` for performance (avoids re-renders during drag). Width is tracked as a percentage (`leftPct`) to remain responsive. The drag handle is a 20px-wide column of 6 dot-grip icons for discoverability.

### 5.6 Soft Delete
Delete actions call `DELETE /api/v1/warehouse/locations/{id}` which triggers a soft delete (`deleted_at` set) on the backend. The frontend filters `deleted_at != null` records out of the displayed list. Hard delete is never exposed.

### 5.7 Go Back Button Placement
The "Go Back" button sits above the split pane container, not inside the left panel header. This keeps it visually independent and always reachable even when the left panel is very narrow.

## 6. API Endpoints Used

| Method | Path | Usage |
|---|---|---|
| GET | `/api/v1/warehouse/locations?warehouse_id={id}` | Fetch all locations for active warehouse |
| POST | `/api/v1/warehouse/locations` | Create single location |
| POST | `/api/v1/warehouse/locations/bulk` | Bulk-create (Pattern Generator commit) |
| PATCH | `/api/v1/warehouse/locations/{id}` | Inline edit update |
| DELETE | `/api/v1/warehouse/locations/{id}` | Soft delete single location |
| POST | `/api/v1/warehouse/locations/bulk-delete` | Soft delete multiple locations |
| POST | `/api/v1/warehouse/locations/bulk-toggle` | Activate/deactivate multiple |

All endpoints require `Authorization: Bearer <token>` (JWT from `AuthContext`).

## 8. Known Limitations & Future Work

- **Pagination on locations list**: currently all locations are fetched in one call; large warehouses (10 000+ locations) will need server-side pagination on the locations endpoint.
- **Undo / redo**: inline edits commit immediately; there is no undo stack.
- **Drag-to-reorder**: `sort_order` is stored but there is no drag-reorder UI yet.
- **CSV column mapping step**: the import wizard validates known column names but does not yet offer a flexible column-remap UI for non-standard headers.
- **Mobile responsiveness**: the split pane is desktop-only; on small screens the generator layout will overflow.
